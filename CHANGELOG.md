# Changelog

Notable changes by release. Tierless is pre-1.0 — a `0.x` minor may break the API.

## 0.2.0 — 2026-07-04

Types you can trust, fixes to shipped bugs, and TypeScript authoring.

- **The declared types now come from the implementation.** The framework's own
  source is TypeScript under `strict`, and every shipped `.d.mts` is generated by
  `tsc` on each build (CI fails on drift). This replaces the hand-written
  declarations 0.1.0 shipped, which had ~20 wrong signatures across five modules
  (`heap`, `delta`, `content`, `fetch`, `graph`) — including a `GLOBALS` typed as
  a `Map` that is really a plain object, and a `content` method that never existed.
- **Fixed: a malformed URL could crash the whole server.** `GET /%` threw an
  uncaught `URIError` out of `serveApp`'s static handler and took the process
  down; it now answers 400. Hostile wire input is hardened the same way: a
  non-numeric bigint on the wire fails with the reader's clean `RangeError`,
  not a raw `SyntaxError`.
- **Fixed: write-back coherence could crash on an ordinary edit.** When
  `min(delta, whole)` picked the whole-snapshot arm, its object ids were
  re-derived over the mutated graph and landed on the wrong master objects —
  a crash, or silent corruption. Both arms now share the fetch-anchored
  baseline (a shape-changing regression proves it).
- **Fixed: a function with no `return` now yields `undefined`**, matching plain
  JS, instead of the internal string `"(end)"`.
- **Write mix modules in TypeScript.** `app.src.ts` (or `.mts`) compiles through
  the same pipeline — erasable TS only, the same ceiling as
  `node --experimental-strip-types` (no enums, no namespaces, no parameter
  properties).
- **`tierless types` emits real signatures.** Each endpoint's parameter list is
  read from its `run: ([sym, n = 1, ...rest]) => …` destructure, so a
  wrong-arity `api.*` call in a mix module fails to type-check; an endpoint
  taking the raw args array keeps the honest `(...args: any[])` fallback. And
  `tierless explain` now rejects exactly what `tierless build` rejects, with a
  clean one-line error.
- One shared, fuzz-tested reader/writer now backs both binary wire formats
  (the formats themselves are unchanged — verified byte-identical).

## 0.1.0 — 2026-07-02

First public release.

```
npm install tierless
npm create tierless@latest my-app
```

Tierless compiles a plain JavaScript function into a state machine that can pause
mid-function, cross the network, and resume on the other side — so a workflow
that touches both a database and the DOM is one function, not a client and a
server glued together with hand-written endpoints. An 8-call workflow that would
normally cost 8 round trips runs in 1.

What's in this release:

- **Compiler.** Lowers ordinary JavaScript — loops, `try`/`catch`/`finally`,
  destructuring, default and rest parameters — into a migratable state machine,
  not a restricted subset. Code that genuinely can't migrate (a tier call inside
  an array callback, for instance) is a build-time error, not a runtime surprise.
- **Wire protocol.** A migrating continuation is typically on the order of
  100–200 bytes and takes low single-digit microseconds to encode; a large
  in-memory value the code never touches crosses as a small handle instead of its
  full contents (thousands of times smaller on a realistic dataset). On a
  long-running session, repeat crossings ship only what changed since the last
  one — commonly 75–90% fewer bytes than resending the full state each time.
- **Trust boundary.** Server-only operations run through a small, separate
  process that authorizes every call against a verified caller — not the
  application code, so it can't be bypassed by tampering with the client. Every
  endpoint must declare who's allowed to call it; there is no default-open path.
- **Tooling.** A CLI (`tierless build` / `explain` / `api` / `types`), a Vite
  plugin with React bindings, and `create-tierless` for a running two-tier app
  in under a minute.
